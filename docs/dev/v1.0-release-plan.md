# php-gui v1.9 Release Plan

A gap analysis of the current state of php-gui and a phased plan to reach a credible production-ready release. The current published version is 1.8; this plan tracks the work toward 1.9. Severity tiers reflect what should block the next ship vs. what can land later.

> **Status:** authored after the Image-widget / nested-Frame work landed. All references to current behaviour reflect the codebase at that point.

## Progress log

### 2026-04-26 — Phase 1.1 landed

System-wide Tcl-safe quoting is now in place.

- `AbstractWidget::tclQuote(string)` — central helper using `"…"` quoting with `\`, `$`, `[`, `"` escaped. Brace-quoting was rejected (trailing-backslash unbalanced-braces case).
- `AbstractWidget::buildOptionString(array $skip)` — replaces every per-widget `getOptionString()` reimplementation; routes every value through `tclQuote()`.
- Updated to use the helpers: `Label`, `Button`, `Checkbutton`, `Input`, `Entry`, `Combobox`, `Menu`, `Menubutton`, `Message`, `Window`, `TopLevel`, `Canvas`, `Image`.
- `Input` / `Entry` `setValue` / `getValue` now go through `ProcessTCL::setVar`/`getVar` (FFI `Tcl_SetVar2`/`Tcl_GetVar2`) instead of string interpolation, bypassing Tcl parsing entirely.
- `TclInjectionTest` (29 assertions) — round-trips escape-attempt payloads through `tclQuote()`, every affected widget, `Combobox` value lists, and `Canvas::drawText`. Asserts the parent window stays alive and values round-trip literally through `cget`.
- `docs/security.md` documents the policy and ties it back to the regression test.
- `docs/_sidebar.md` links the new page from the docs site.
- Test totals: **188 assertions across 16 files, all passing** (was 159/15 before this phase).

Remaining for Phase 1: 1.2 (event-loop callback bridge), 1.3 (callback lifecycle), 1.4 (window event hooks).

### 2026-04-26 — Phase 1.2 / 1.3 / 1.4 landed

The remainder of Phase 1 is in. Phase 1 is now complete.

**1.2 — In-process callback bridge.** Replaced the `/tmp/phpgui_callback.txt` polling with a Tcl-list queue:

- `proc php::executeCallback {id} { lappend ::phpgui_pending $id }`
- `ProcessTCL::drainPendingCallbacks()` — reads `::phpgui_pending`, clears it, dispatches each id in FIFO order. Returns count for tests.
- `ProcessTCL::shouldQuit()` reads `::phpgui_quit` (set by `::exit_app`); the temp-file quit signal is gone too.
- `Application::tick()` calls `update`, then `drainPendingCallbacks`, then `shouldQuit`. Loop sleep dropped from 100ms to 10ms (~100Hz responsiveness).
- Lost-callback bug fixed: the queue holds an arbitrary number of ids, so two events fired between ticks both dispatch.
- Cross-process collision fixed: no shared filesystem state.

**1.3 — Callback lifecycle.**

- `ProcessTCL::unregisterCallback(string $id)` removes both the PHP closure and the Tcl-side `php::callbacks($id)` entry.
- `AbstractWidget::destroy()` calls `unregisterCallback($this->id)` automatically — covers Button, Checkbutton, Input::onEnter.
- `Menu::destroy()` overrides to free its per-command callback ids (`{menuId}_cmd_0`, `_cmd_1`, …) and recursively destroy its submenus.
- A queued callback id that has been deregistered is a safe no-op (skipped by `executeCallback`).

**1.4 — Window event hooks.**

- `Window::onClose(callable)` — runs handler before exit. Returning `false` vetoes the close (window stays open). Auxiliary callback id `{windowId}_close`.
- `Window::onResize(callable)` — fires with `(int $width, int $height)`. Filters `<Configure>` events that originate from descendants and ignores duplicate-size events. Auxiliary callback id `{windowId}_resize`.
- `Window::destroy()` overrides to free both auxiliary ids.
- `Application::quit()` no longer calls `exit(0)` — flips the run flag and tears down WebViews, then `run()` returns to the caller.

**Tests added.**

- `tests/widgets_test/EventLoopTest.php` (19 assertions): queue ordering, drain behaviour, `Application::tick()` integration, quit signal, callback unregister-on-destroy for both Button and Menu, plus assertions that the old temp files are no longer produced.
- `tests/widgets_test/WindowEventsTest.php` (13 assertions): `onClose` runs and triggers `::exit_app` by default, returning `false` vetoes, `onResize` reports new dimensions, both handlers are freed by `destroy()`.

**Docs added.**

- `docs/event-loop.md` — explains the new IPC, how to drive the loop in tests, the quit signal, and the callback-lifecycle contract for custom widgets.
- `docs/Window.md` — added `onClose` and `onResize` to the methods table with usage examples.
- `docs/_sidebar.md` — links the new event-loop page.

**Test totals after Phase 1: 220 assertions across 18 files, all passing** (was 188/16).

## What's already solid

- Cross-platform native rendering through PHP FFI to Tcl/Tk; Linux libs are bundled and rebuilt against an older glibc (works on glibc 2.34+).
- Two render modes — Tcl/Tk widgets and a WebView helper subprocess — share a single event loop in `Application::run()`.
- 159 assertions across 15 test files (`tests/index_test.php` + `tests/widgets_test/*`).
- The `Image` widget is in production shape: animated GIF playback with disposal compositing, GD-transcoded JPEG/BMP, safe Tcl quoting, temp-file cleanup, deeply tested.
- Arbitrary widget nesting works (Window → Frame → Frame → Button) after the `AbstractWidget` path-tracking refactor.

## Gap inventory

### Critical — ship-blockers

These must be fixed before 1.0.

#### C1. System-wide Tcl injection
Every widget except `Image` interpolates user-supplied strings (`text`, option values) directly into Tcl commands using `\"$value\"`. A label whose text is `Hello"; destroy .; "` will execute the embedded `destroy .` command.

Affected files:
- `src/Widget/Label.php` — `text`, `font`, `fg`, `bg`, `state`, all option keys
- `src/Widget/Button.php` — `text` and option values
- `src/Widget/Checkbutton.php` — `text` and option values
- `src/Widget/Input.php` — `text` (default value), all option keys
- `src/Widget/Entry.php` — `text` (default value)
- `src/Widget/Combobox.php` — `values`
- `src/Widget/Menu.php` — `label` arguments in `addCommand`, `addSubmenu`, option values
- `src/Widget/Menubutton.php` — `text`
- `src/Widget/Message.php` — `text`
- `src/Widget/TopLevel.php` — `title`, all option keys
- `src/Widget/Window.php` — `title`
- `src/Widget/Canvas.php` — `text` in `drawText`, all option values

Image already uses the safe pattern — adapt that across the others.

#### C2. Event-loop callback bridge is fragile
`Application::tick()` polls `/tmp/phpgui_callback.txt` every 100 ms. Three concrete failures:

- **Lost callbacks.** Two clicks within a tick collapse to the latest ID; the earlier one is dropped.
- **100 ms input lag** on every callback — feels noticeably sluggish.
- **Cross-process collision.** Any two php-gui apps running on the same host share `/tmp/phpgui_callback.txt` and route callbacks to the wrong process.

The quit signal (`/tmp/phpgui_quit.txt`) shares the second and third failure modes.

#### C3. Callback registry leaks
`ProcessTCL::registerCallback()` adds closures to a static array; `AbstractWidget::destroy()` never removes them. Long-running apps that create and discard widgets accumulate closures and their captured state forever. The Image animation keeps a frame-photo registry that has the same issue at a smaller scale.

#### C4. Window close is hard-wired to `exit_app`
`Window::create()` sets `WM_DELETE_WINDOW` to `::exit_app` with no application hook. There is no way to intercept the close event for an "unsaved changes — really quit?" prompt. `Application::quit()` itself calls `exit(0)`, preventing graceful shutdown by callers.

### High — essential widgets the library is missing

Any non-trivial desktop app needs these. Tk provides them; we just don't wrap them yet.

| Widget | Tk command | Why it matters |
|---|---|---|
| `Text` | `text` | Multi-line text editor — notes, logs, code, descriptions |
| `Scrollbar` | `scrollbar` / `ttk::scrollbar` | Without it, `Text`/`Listbox`/`Canvas` are unusable for real content |
| `Listbox` | `listbox` | Selection lists |
| `Radiobutton` | `radiobutton` | Mutually-exclusive choices via shared variable |
| `Scale` | `scale` | Slider input |
| `Spinbox` | `ttk::spinbox` | Bounded numeric input |
| `Progressbar` | `ttk::progressbar` | Feedback for long-running tasks |
| `Notebook` | `ttk::notebook` | Tabbed UIs — table-stakes for settings, multi-pane apps |
| `Treeview` | `ttk::treeview` | Tables and hierarchical lists. **The single biggest missing piece** for desktop apps |
| `PanedWindow` | `panedwindow` / `ttk::panedwindow` | Resizable splits (file tree + editor) |
| `LabelFrame` | `labelframe` / `ttk::labelframe` | Titled group box |
| `Separator` | `ttk::separator` | Visual divider |

### Medium — API completeness

5. **No generic `configure(array)` / `cget(string)` on `AbstractWidget`.** Each widget hand-rolls a few setters (`Label` has 5; `Button` has 0). Users can't change a Button's `bg` or `text` after creation without dropping to raw `evalTcl`.

6. **No general `bind(event, handler)` API.** Only `Button::command` and `Input::onEnter` are wired. No way to add `<Control-s>` shortcuts, `<Button-3>` right-clicks, hover handlers.

7. **No `gridConfigure(col/row, weight, uniform)` helper.** `example.php` had to drop into raw `evalTcl()` to get equal-weighted columns. Should be exposed on every widget.

8. **No `pack_forget` / `grid_forget` / `place_forget`** for hide-but-keep-state.

9. **No clipboard API.** Tk's `clipboard` command isn't exposed.

10. **No window icon API.** `wm iconphoto` isn't exposed.

11. **No `getText()` / state getters** on most display widgets. `Label::setText()` exists but no `getText()`.

---

### Low — infrastructure & hygiene

12. **No CI.** No `.github/workflows/`. With 159 assertions and no automated runs across Linux/macOS/Windows, regressions land unnoticed.
13. **No PHPStan/Psalm config.** Static analysis would catch several issues above.
14. **Stale docs.** `docs/Frame.md`'s nesting example didn't actually work until the recent refactor; other doc examples may be similarly aspirational and should be cross-referenced against tests.
15. **Mixed test fixtures.** `tests/widgets_test/image/example.jpg` is committed but only used by the example app, not by tests.
16. **No documented build for WebView helper binaries** (`webview_helper`, `webview_helper_linux_x86_64`, Windows variant). They're checked in without a reproducible build path.
17. **No HiDPI documentation.** Tk's `tk scaling` works but isn't surfaced or documented.
18. **No Unicode/multibyte input integration test.** Tcl 8.6 is UTF-8 internally, but the PHP↔Tcl FFI string round-trip should be verified explicitly for non-ASCII input.

## Phased plan

### Phase 1 — Ship-blockers (~3–5 days)

**1.1 — System-wide Tcl-safe quoting**
- Add a static helper to `AbstractWidget`:
  ```php
  protected static function tclQuote(string $value): string {
      $escaped = str_replace(['\\', '$', '[', '"'], ['\\\\', '\\$', '\\[', '\\"'], $value);
      return "\"{$escaped}\"";
  }
  ```
- Replace every `\"$text\"` and `-key \"$value\"` interpolation across the affected widgets (see C1).
- Where a value is already routed through a Tcl variable (the pattern Image uses for paths), keep that — it's even safer than quoting.
- Add a regression test: a label whose text contains `"; destroy .; "` must display literally and not destroy the window.

**1.2 — Replace temp-file callback IPC with an in-process Tcl variable**
- Change the Tcl `php::executeCallback` proc from "write file" to "lappend to global":
  ```tcl
  proc php::executeCallback {id} { lappend ::phpgui_pending $id }
  ```
- In `Application::tick()`: read `::phpgui_pending` via `getVar`, dispatch each ID, then `set ::phpgui_pending {}`. No filesystem, no race, no cross-process collision, callbacks queue correctly.
- Apply the same shape to the quit signal (`::phpgui_quit` Tcl variable, not a file).
- Drop `usleep(100000)` to `usleep(10000)` — feels native, still cheap.

**1.3 — Callback lifecycle tied to widget lifecycle**
- Add `ProcessTCL::unregisterCallback(string $id)`.
- `AbstractWidget::destroy()` calls it for `$this->id` (and any extra IDs the widget registered, e.g. Menu uses `id_cmd_N`).
- When a parent is destroyed, walk the widget registry and unregister descendants too.

**1.4 — Window event hooks**
- `Window::onClose(callable $handler)` / `TopLevel::onClose(callable $handler)` that intercept `WM_DELETE_WINDOW`. If the handler returns `false`, the window stays open.
- `Window::onResize(callable $cb)` binding to `<Configure>` (debounced).
- `Application::quit()` should not call `exit()` itself — let the caller decide.

### Phase 3 — API completeness (~2–3 days)

- `AbstractWidget::configure(array $options)` / `cget(string $option)` — any Tk option readable/writable generically.
- `AbstractWidget::bind(string $event, callable $handler)` — registers a callback under a derived ID, binds the event, deregisters on destroy. Same plumbing as Button's `command`.
- `AbstractWidget::columnConfigure(int $col, array $opts)` / `rowConfigure(int $row, array $opts)`.
- `pack_forget()` / `grid_forget()` / `place_forget()`.
- `Clipboard::set($text)`, `Clipboard::get()`, `Clipboard::clear()`.
- `Window::setIcon($path)` — wraps `wm iconphoto`.

### Phase 4 — Infrastructure & hygiene (~1–2 days)

- **GitHub Actions CI**: matrix of every test file on `ubuntu-latest`, `macos-latest`, `windows-latest`. PRs blocked on test failures.
- **PHPStan level 6** on `src/` (or Psalm). Settle on a level and enforce.
- **Doc audit**: cross-reference each `docs/*.md` example against a working test or example. Where the doc shows untested code, either add a test or fix the doc.
- **Move `tests/widgets_test/image/example.jpg`** out of test fixtures — it's an example asset, not a test fixture.
- **Document the WebView helper build** so contributors can rebuild it per platform.
- **Add a Unicode integration test** for `Input`/`Entry`/`Label` covering CJK + emoji + RTL.

## Recommended v1.0 scope

For a credible 1.0:

- **All of Phase 1** (the ship-blockers).
- **Phase 2 widgets 1–7** (Text, Scrollbar, Listbox, Radiobutton, Scale, Spinbox, Progressbar). Notebook + Treeview can land in 1.1 if scope pressure demands it, but every additional one expands the realistic app surface.
- **Phase 3** items `configure` / `bind` / `columnConfigure` (the rest can trail).
- **Phase 4** CI and the doc audit.

Estimate: **~10 working days** for a single contributor.

Without the Tcl-injection fix (C1) and the event-loop fix (C2), the library is not safe in front of users handling non-trivial input. Without `Text` + `Scrollbar` + `Notebook` + `Treeview`, the library is stuck being a demo.
